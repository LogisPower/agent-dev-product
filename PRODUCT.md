# Product Definition

## Purpose
This product exists to prove that an automated agent team can fully complete software features and improvements.

## Definition of Done
A task is complete when:
- Acceptance criteria are met
- Code passes tests (if applicable)
- No forbidden areas are modified
- A pull request is approved and merged

## Constraints
- No direct commits to main
- All work must be traceable to a GitHub issue
- Changes must be incremental and reviewable

## Non-Goals
- Large rewrites without approval
- Infinite refactoring loops
